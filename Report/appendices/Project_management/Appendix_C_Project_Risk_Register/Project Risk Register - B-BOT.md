## Review Notes

The main risk response during the project was scope control. The original proposal was broader and expected more direct ROS/micro-ROS integration with the robot controller. As the embedded balance and safety work became the dominant technical risk, the final architecture deliberately kept the ESP32 balance loop local and used ROS 2 vision only as a supervisory command source.

By the final report stage, the highest delivery risk had shifted from missing data to reproducibility and scope control. The E1-E11 datasets are now measured and the report separates the remaining limitations from claimed achievements. The main residual management control is to keep the submitted GitHub repository public, frozen at the submitted report state, and consistent with the appendix evidence.
