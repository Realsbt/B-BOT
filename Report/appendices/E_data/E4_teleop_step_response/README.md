# E4 Teleop Step Response

Date/time: 2026-04-24 UK test session; filename/date stamp generated by the local VM/computer
Controller mode: ESP32 TCP command parser, direct teleoperation commands
Hardware state: ESP32 controller board reachable over WiFi; full robot used for E4b physical-response measurements
Environment: local hotspot, host PC to ESP32 TCP port `23`
Trial list: see `tcp_step_response_2026-04-24.csv` for E4a; see `physical_step_trials_2026-04-24.csv` and `physical_step_summary_2026-04-24.csv` for E4b
Safety notes: every step trial ends with `DRIVE,0,0`, `YAWRATE,0`, and `QUEUE_STOP`.

Status: MEASURED (E4a command-entry; E4b physical response)

Purpose:

- Quantify how quickly the robot responds to teleoperation step commands.
- Report speed/yaw rise time, steady-state error, and pitch disturbance during commanded motion.
- E4a measures command-entry step response only: host command to ESP32 ACK.
- E4b measures physical wheel/body response on the full robot.
